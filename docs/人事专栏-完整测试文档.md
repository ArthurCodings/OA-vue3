## 一、测试准备

### 1.1 环境准备

| 项目 | 检查项 | 预期 | 备注 |
|------|--------|------|------|
| 后端服务 | `yudao-server` 已启动 | 端口正常监听 | 默认 48080 |
| 前端服务 | `yudao-ui-admin-vue3` 已启动 | 可访问登录页 | 默认 80 或 3000 |
| 数据库 | MySQL 已启动，11 张表已创建 | 无建表报错 | 按开发指南顺序建表 |
| 定时任务 | Quartz/Spring Scheduled 已启用 | 排班生成、考勤计算定时任务可执行 | 检查 `@EnableScheduling` |
| 文件存储 | `infra/file/upload` 可用 | 上传返回 URL | 用于文档、合同附件 |
| 租户 | 若为多租户，确认当前租户 ID | 测试数据在正确租户下 | 默认 tenant_id=1 |

**验证命令示例**：
```bash
# 检查后端
curl -s http://localhost:48080/admin-api/system/attendance-rule/list-enabled -H "Authorization: Bearer {token}"

# 检查前端
# 浏览器访问 http://localhost 或配置的端口
```

---

### 1.2 账号与权限准备

| 角色 | 用途 | 需分配权限 |
|------|------|------------|
| 超级管理员 | 全功能测试 | 所有权限 |
| HR 管理员 | 考勤/薪资/花名册管理 | `system:attendance-*`, `system:salary-*`, `system:hr-employee-*` |
| 部门负责人 | 数据权限测试 | 考勤/薪资「本部门」数据权限 |
| 普通员工 | 打卡、个人中心 | `system:attendance-record:clock`，无管理权限 |
| 无花名册用户 | 边界测试 | 仅登录，无花名册记录 |

**准备步骤**：
1. 系统管理 → 角色管理：创建「HR 管理员」「普通员工」等角色
2. 为各角色分配对应权限（参考开发指南 5.1 权限标识一览）
3. 系统管理 → 用户管理：创建测试用户并分配角色
4. 确保至少 2 个在职员工（用于排班、考勤、薪资测试）

### 1.3 基础数据准备（测试前必做）

按以下顺序准备，否则后续测试会失败：

| 序号 | 数据 | 说明 | 验证方式 |
|------|------|------|----------|
| 1 | 排班规则 | 至少 1 条早班、1 条晚班且 status=0 | 考勤 → 排班规则，列表有数据 |
| 2 | 法定节假日 | 当前年至少录入若干天（含休息日、调班日） | 考勤 → 法定节假日 |
| 3 | 花名册 | 至少 2 名在职员工（employment_status=1） | 人事专栏 → 花名册 |
| 4 | 员工薪资档案 | 为上述员工创建薪资档案 | 薪资结算 → 员工薪资档案 |
| 5 | 薪资配置 | 一条有效配置（全勤奖等） | 薪资结算 → 薪资配置 |
