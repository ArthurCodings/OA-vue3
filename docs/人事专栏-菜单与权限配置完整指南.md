# 人事专栏 & 基础设施 — 菜单与权限配置完整指南

> 基于若依 RuoYi-Vue 框架，说明目录、菜单、按钮的细化配置，以及查看/增删改等权限的分离方式。

---

## 一、若依菜单类型说明

若依系统中，菜单分为三种类型，形成「目录 → 菜单 → 按钮」的层级结构：

| 类型 | 说明 | 是否显示在侧边栏 | 用途 |
|------|------|------------------|------|
| **目录** | 一级分组 | 是 | 将多个菜单归类，如「人事专栏」「基础设施」 |
| **菜单** | 可点击进入的页面 | 是 | 对应具体页面路由，如「排班规则」「花名册」 |
| **按钮** | 页面内的操作权限 | 否（通常隐藏） | 控制页面内按钮显示，如「新增」「修改」「删除」 |

**权限分离原则**：
- **查看（query）**：控制能否进入页面、调用列表/详情接口
- **增删改（update/create/delete）**：控制新增、修改、删除按钮及对应接口
- **导出（export）**：控制导出按钮及导出接口
- **特殊操作**：如「生成排班」「发起结算」「查看身份证」等，单独配置

## 四、个人中心聚合面板

### 4.1 访问方式

个人中心为**固定路由**，不通过菜单管理配置：

- **路由**：`/user/profile`
- **入口**：右上角头像 → 个人中心
- **工作台 Tab**：个人中心内的「工作台」Tab 展示聚合面板

### 4.2 聚合面板内容

| 卡片 | 数据来源 | 说明 |
|------|----------|------|
| 今日打卡 | `/system/personal-center/clock-in`、`clock-out` | 签到/签退，需 `system:attendance-record:clock` |
| 本月考勤 | `/system/personal-center/panel` | 应出勤/实出勤/迟到/缺勤 + 日历 |
| 待确认薪资通知单 | `/system/salary/my-confirm-notices`、`my-confirm` | 员工确认薪资，仅需登录 |
| 最新薪资单 | panel 或 `/system/salary/my-latest` | 仅展示已确认/已发放 |
| 我的绩效 | `/system/performance/my-score`、`my-appeal` | 查看绩效、发起申诉，仅需登录 |
| 员工档案 | panel | 本人档案 |
| 我的合同 | panel | 本人负责的合同 |

### 4.3 权限说明

- **panel 接口**：仅需登录，无需额外权限
- **打卡按钮**：需分配 `system:attendance-record:clock`

### 4.4 个人中心相关接口（无需菜单权限）

以下接口只要求登录，不校验菜单权限，通常放在个人中心页面：

| 接口 | 说明 |
|------|------|
| `GET /system/salary/my-latest` | 最新薪资单 |
| `GET /system/salary/my-list` | 历史薪资列表 |
| `GET /system/salary/my-confirm-notices` | 待确认薪资通知 |
| `PUT /system/salary/my-confirm` | 员工确认薪资 |
| `GET /system/performance/my-score` | 本人绩效打分 |
| `PUT /system/performance/my-appeal` | 绩效申诉 |

**说明**：这些接口不需要在菜单管理中配置权限，角色分配时也无需勾选。

## 五、在菜单管理中的具体操作

1. 在「人事专栏」下直接新增各菜单：薪资配置、员工薪资档案、月度薪资、绩效模板、绩效打分、花名册等（见第三节）
2. 为每个菜单新增对应按钮（见 3.6～3.11 节）
3. 在「系统管理 → 角色管理」中，为 HR、财务等角色勾选上述菜单和按钮，保存后即可生效

## 六、角色分配建议

在 **系统管理 → 角色管理** 中，按角色勾选菜单和按钮：

| 角色 | 建议分配的菜单/权限 |
|------|----------------------|
| 超级管理员 | 全部（*:*:* 或勾选全部） |
| HR/人事 | 人事专栏全部（排班、考勤、薪资配置、员工薪资档案、月度薪资、绩效模板、绩效打分、花名册）+ 打卡 |
| 财务 | 薪资配置、员工薪资档案、月度薪资 + 打卡 |
| 绩效审核人 | 绩效模板（查询）、绩效打分（查询+维护+审核） |
| 部门负责人 | 考勤排班、打卡记录（查看）、花名册（查看）、打卡 |
| 普通员工 | 打卡、个人中心工作台 |

## 七、配置要点总结

1. **查看与增删改分离**：`query` 控制进入页面，`update` 控制增删改按钮。
2. **按钮类型菜单**：类型选「按钮」，父菜单选对应页面，勾选「隐藏」。
3. **权限字符一致**：菜单中的权限字符需与前端 `v-hasPermi`、后端 `@PreAuthorize` 完全一致。
4. **导出单独配置**：`export` 权限单独控制导出按钮。
5. **敏感字段**：花名册的 `sensitive` 控制身份证明文显示。
6. **路由地址唯一**：同一父菜单下，各子菜单的「路由地址」必须唯一，不可重复，否则会导致路由冲突、页面无法正确显示。

## 七（附）、常见问题与故障排查

### 员工薪资档案无法显示 / 点击后显示花名册内容

**原因**：员工薪资档案与花名册使用了相同的「路由地址」`employee`，导致路由冲突。同一父菜单下，Vue Router 会按 path 匹配，两个菜单 path 相同时只会匹配到其中一个。

**解决**：在 **系统管理 → 菜单管理** 中，找到「员工薪资档案」菜单，将「路由地址」从 `employee` 改为 `salary-employee`，「组件路径」保持为 `hr/salary/employee/index` 不变。修改后重新登录或刷新页面即可生效。
